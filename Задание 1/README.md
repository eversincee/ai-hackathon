# Система автоматизации оцифровки паспортов оборудования

**Хакатон РТУ МИРЭА 2026 — Задача 1**

---

## Проблема и подход к решению

Паспорта оборудования (шкафы КИПиА, ГПА, насосные агрегаты) существуют в бумажном виде. На приёмку шкафа из 26 единиц уходит несколько часов: оператор вручную переносит данные с каждого паспорта в 1С. Ошибки неизбежны — нечёткий шрифт, усталость, разные форматы у разных производителей.

### Почему двойное распознавание (ensemble), а не просто OCR

Классический подход — OCR → шаблон → в базу — ломается на реальных паспортах: страница повёрнута, печать перекрывает текст, у TREI и TECON разные форматы полей. Чтобы не делать отдельный шаблон под каждого производителя и при этом сохранить высокую точность, мы запускаем **две независимые ветки** параллельно:

| Ветка | Что делает | Сильные стороны |
|-------|-----------|-----------------|
| **VLM** (Qwen2.5-VL) | Нейросеть «смотрит» на изображение и сразу выдаёт JSON | Понимает структуру таблиц и расположение полей, устойчива к плохому качеству скана |
| **OCR → LLM** (Surya + Qwen2.5) | Surya читает текст со скана, Qwen2.5 структурирует его в JSON | Точнее на печатном тексте высокого качества, меньше галлюцинаций на числах |

Затем результаты **сливаются** по правилам:
- Оба дали одинаковое значение → уверенность 100%, поле подсвечивается зелёным
- Один пустой, второй нет → уверенность 70%, оператор видит предупреждение
- Значения расходятся → уверенность 50%, оператор выбирает нужное вручную

Это даёт систему, которая **сама знает, чему доверять** — вместо того чтобы молча выдавать неверные данные.

### Почему локальные модели, а не облако

Паспорта оборудования часто содержат коммерческую и техническую тайну. Отправлять их в облако — риск. Qwen2.5-VL 3b и Qwen2.5 7b запускаются локально через Ollama и не требуют GPU — достаточно обычного ноутбука с 8 ГБ RAM.

### Что система умеет сверх простого OCR

- **Валидация** — проверяет формат номера документа, соответствие серийников паттернам производителя (ТРИ/TECON), наличие даты выпуска при указанной гарантии
- **Штрихкоды Code 128** — каждый паспорт получает уникальный код, PDF-этикетку можно распечатать и наклеить на бумажный оригинал
- **Сборки** — группировка 26 паспортов шкафа в одну запись с общим штрихкодом и таблицей в формате Приложения 4
- **Экспорт в 1С** — CSV с BOM (открывается без кракозябр) и CommerceML XML
- **Чек-лист PDF** — для приёмки на складе без компьютера

## Как это работает (пошагово)

```
1. ЗАГРУЗКА
   Пользователь перетаскивает PDF в браузер (страница "Загрузить паспорта")
                    ↓
2. РАЗРЕЗКА
   Backend разрезает PDF на отдельные страницы-картинки (PyMuPDF)
                    ↓
3. АВТО-ПОВОРОТ
   Если страница повёрнута — поворачиваем в правильное положение
                    ↓
4. ДВОЙНОЕ РАСПОЗНАВАНИЕ (ensemble)
   Параллельно запускаются две независимые ветки:
   
   Ветка A: VLM (Qwen2.5-VL)              Ветка B: OCR → LLM
   Нейросеть СМОТРИТ на картинку       Surya ЧИТАЕТ текст со скана
   и сразу выдаёт JSON с полями    →   затем Qwen2.5 СТРУКТУРИРУЕТ
                                        этот текст в JSON
                    ↓
5. ГОЛОСОВАНИЕ
   Сравниваем результаты двух веток:
   - Оба согласны → высокая уверенность (зелёное поле)
   - Расхождение → низкая уверенность (жёлтое/красное поле)
   - Один пуст → берём непустой (средняя уверенность)
                    ↓
6. ВАЛИДАЦИЯ
   Проверяем извлечённые данные:
   - Номер документа соответствует известному формату? (БНРД.XXX.XXX ПС)
   - Серийники соответствуют паттерну производителя?
   - Производитель совпадает с префиксом номера документа?
   - Если гарантия указана — есть ли дата выпуска?
                    ↓
7. МАРШРУТИЗАЦИЯ
   - Всё ок + уверенность > 90% → автоматически одобрено
   - Есть проблемы → отправляется на ручную проверку
                    ↓
8. РЕВЬЮ (если нужно)
   Оператор видит: слева скан, справа форма с полями
   Поля подсвечены по уверенности (зелёный/жёлтый/красный)
   Клик по полю → подсвечивается место на скане, откуда оно взято
   Оператор правит ошибки → нажимает "Одобрить"
                    ↓
9. ШТРИХКОД
   Система генерирует Code 128 с уникальным ID паспорта
   Можно скачать PDF-этикетку и наклеить на бумажный оригинал
                    ↓
10. ЭКСПОРТ В 1С
    Одобренные паспорта экспортируются:
    - CSV (для загрузки в "Фактическое Поступление")
    - CommerceML XML (стандарт обмена 1С)
    - Чек-лист PDF (для приёмки на складе)
```

### Что такое «Сборка»

Сборка — это группа паспортов, объединённых под одним изделием. Например, шкаф САУ ГПА содержит 26 единиц оборудования, у каждой свой паспорт. Пользователь:
1. Создаёт сборку (указывает название, код проекта, заводской номер)
2. Выбирает паспорта из списка оцифрованных
3. Система генерирует общий штрихкод на сборку и таблицу в формате Прил. 4

### Что такое «Импорт 1С»

В 1С уже есть документ «Фактическое Поступление» с номенклатурой. Пользователь экспортирует его в CSV, загружает в нашу систему → при ревью паспорта можно привязать его к конкретной строке 1С через «Код номенклатуры СУ». Это связывает бумажный паспорт с цифровой записью.

## Архитектура (4 сервиса)

```
Браузер (localhost:3000)          ← Next.js + React + Tailwind
        ↕ HTTP
Backend (localhost:8000)          ← FastAPI + SQLite
        ↕ HTTP                    
OCR-сервис (localhost:8001)       ← Surya (распознавание текста)
Inference (localhost:11434)       ← Ollama (Qwen2.5-VL + Qwen2.5)
```

Каждый сервис запускается и останавливается независимо. Inference при необходимости можно вынести на машину с GPU — меняется одна строчка в `.env`.

## Быстрый старт

### Что нужно на машине

| Программа | Зачем | Как поставить |
|-----------|-------|---------------|
| Python **3.11 или 3.12** | Backend + OCR | macOS: `brew install python@3.12`. Windows: python.org |
| Node.js 20+ | Frontend | `brew install node` или nodejs.org |
| Ollama | Запуск нейросетей | ollama.com/download |

Плюс свободно: **15 ГБ на диске** (модели Ollama + Surya) и **8 ГБ RAM** (с `qwen2.5vl:3b`). Порты `3000`, `8000`, `8001`, `11434` должны быть свободны.

### Шаг 1: Скачать модели (нужен интернет, ~7 ГБ)

```bash
# Запустить Ollama (или открыть приложение Ollama.app)
ollama serve

# В другом терминале — скачать модели:
ollama pull qwen2.5vl:3b          # VLM для чтения картинок (~2 ГБ)
ollama pull qwen2.5:7b-instruct   # LLM для структурирования текста (~5 ГБ)
```

### Шаг 2: Запустить OCR-сервис

```bash
cd ocr_service
python3.12 -m venv .venv
source .venv/bin/activate        # Linux/Mac
# .venv\Scripts\activate         # Windows (PowerShell)
pip install -e .
uvicorn main:app --port 8001
```

Первый раз Surya подтянет ~3 ГБ своих моделей с Hugging Face — это 5–10 минут, зависит от интернета. Дальше стартует мгновенно.

### Шаг 3: Запустить Backend

```bash
cd backend
python3.12 -m venv .venv
source .venv/bin/activate        # Linux/Mac
# .venv\Scripts\activate         # Windows (PowerShell)
pip install -e ".[dev]"
cp ../.env.example .env          # настройки по умолчанию
uvicorn app.main:app --port 8000 --reload
```

### Шаг 4: Запустить Frontend

```bash
cd frontend
npm install    # только при первом запуске
npm run dev    # http://localhost:3000
```

### Шаг 5: Открыть браузер

Заходим на **http://localhost:3000** — видим главную страницу с тремя кнопками.

Проверить что все сервисы живы:

```bash
curl http://localhost:8000/health      # {"status":"ok"}
curl http://localhost:8001/health      # {"status":"ok","languages":["ru","en"]}
curl http://localhost:11434/api/tags   # список моделей Ollama
```

## Демо-сценарий (5 минут)

1. **Главная** (30 сек) — показать dashboard, счётчики, три кнопки
2. **Загрузка паспорта** (60 сек) — перетащить PDF из `Задание 1/`, показать прогресс обработки
3. **Ревью** (90 сек) — открыть обработанный паспорт, показать форму с цветными полями, отредактировать поле с низкой уверенностью, одобрить
4. **Сборка** (60 сек) — создать сборку из нескольких паспортов, показать сгенерированный штрихкод
5. **Сканирование** (30 сек) — ввести код штрихкода → перейти на карточку
6. **Экспорт** (30 сек) — скачать CSV, открыть в Excel, показать колонки 1С

## Технологии

| Что | Зачем |
|-----|-------|
| **FastAPI** (Python) | Backend-сервер, принимает запросы от браузера |
| **SQLite** | База данных — один файл `passports.db` |
| **Surya** | Распознавание текста на сканах (OCR), русский язык |
| **Qwen2.5-VL** | Нейросеть, которая «смотрит» на картинку и извлекает данные |
| **Qwen2.5-Instruct** | Текстовая нейросеть для структурирования OCR-текста |
| **Ollama** | Программа для запуска нейросетей локально |
| **Next.js + React** | Веб-интерфейс в браузере |
| **shadcn/ui + Tailwind** | Красивые компоненты и стили |
| **python-barcode** | Генерация штрихкодов Code 128 |
| **reportlab** | Генерация PDF-этикеток и чек-листов |
