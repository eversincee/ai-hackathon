# Модуль диагностики сигнала датчиков

Модуль на C99 для предиктивного анализа сигналов аналоговых датчиков газотурбинного двигателя в реальном времени. Выявляет фактические и потенциальные отказы датчиков, выдавая качественный флаг на каждый отсчёт каждого канала:

- `0` — **Bad** (фактический отказ: резкий скачок, залипание, всплеск шума)
- `1` — **Good** (сигнал в норме)
- `2` — **Potential** (потенциальный отказ: устойчивое смещение или медленный дрейф)

## Метод диагностики

Самообучение на первых 250 отсчётах (≈5 сек при 50 Гц), затем параллельная работа четырёх детекторов:

| Детектор | Что ловит | Флаг |
|---|---|---|
| Темп (rate-of-change) | Резкие скачки, импульсы | Bad |
| Скользящая дисперсия | Залипание, всплески шума | Bad |
| CUSUM с затуханием | Накопленное смещение | Potential |
| EWMA | Медленный дрейф | Potential |

## Требования

- GCC (любая версия с поддержкой C99)
- `make`
- Библиотека `libm` (входит в стандартный Linux)

Для кросс-компиляции дополнительно:
- `arm-linux-gnueabihf-gcc` — для ARM
- `mips-linux-gnu-gcc` — для MIPS

## Установка и сборка

```bash
# Клонировать или скопировать папку sensor_diag на целевую машину
cd sensor_diag

# Сборка для x86 (нативно)
make x86

# Кросс-компиляция для ARM
make arm

# Кросс-компиляция для MIPS
make mips

# Запуск тестов
make test
```

## Запуск

```bash
./sensor_diag_x86 <входной_файл.csv> <префикс_выходных_файлов>
```

**Пример:**
```bash
./sensor_diag_x86 "Набор данных 1/1.csv" output_1
```

Создаёт по одному CSV-файлу на каждый канал датчика:
```
output_1_01.csv  — канал 1
output_1_02.csv  — канал 2
...
output_1_23.csv  — канал 23
```

## Формат выходного файла

```
TimeStamp;Имя датчика;0 - Bad, 1 - Good, 2 - Potential
12:10:00.001;21.548;1
12:10:00.024;21.548;1
...
```

- Разделитель: `;`
- Строки: CRLF (Windows)
- Кодировка: CP1251

## Ресурсы

| Параметр | Значение | Лимит |
|---|---|---|
| RAM на канал | 0.5 КБ | 256 КБ |
| RAM общая | ~50 КБ | 5 МБ |
| Время одного цикла | ~4 мкс | 20 мс |
| `malloc` / `calloc` | не используется | запрещено |

## Поддерживаемые форматы входных данных

- Разделитель полей: `;`
- Десятичный разделитель: `.` или `,` (оба поддерживаются)
- Кодировка: CP1251
- Первая строка — заголовок (пропускается)
- Первый столбец — `TimeStamp`, остальные — значения датчиков
